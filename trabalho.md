Trabalho AABD

Neste trabalho, iremos exercitar como modelar dados não relacionais e como utilizar um
SGBD noSQL. Este trabalho poderá ser feito em grupos de, no máximo, 3 alunos. O
trabalho será feito seguindo os sete passos abaixo:
1) Você deve escolher um par de SGBD noSQL que deseja estudar o funcionamento.
Para esse par, você deverá ter, obrigatoriamente, um SGBD baseado em grafos e
um baseado em documentos. Abaixo estão os exemplos de SGBD que você pode
usar.
Grafos:
Baseados em Cypher: neo4j ou redisGraph;
Baseados em SPARQL: GraphDB ou BlazeGraph
Documentos: Apache Solr, ElasticSearch ou mongoDB
2) Você deve escolher um exemplo de aplicação, a qual você irá modelar e criar os
dados. A aplicação deve ser uma das que estão no seguinte link
https://gist.github.com/jairofsouza/09e8df8a4ee43bf31f583360ff4e7c9b.
Informe ao professor pelo classroom qual tema seu grupo escolheu. Teremos
apenas 1 grupo por tema, assim, ao final das escolhas, teremos no máximo 10 ou
15 grupos formados. Novos temas serão adicionados se houver necessidade...
3) Você deverá descrever um conjunto de dados que estarão presentes no banco de
dados e criar uma modelagem desses dados. Nesta disciplina, não teremos o rigor
que se tem com a modelagem em DCC060, então apenas descreva seus dados
principais que seriam usados na aplicação que você escolheu e crie um conjunto
de tuplas para popular esse dado. É necessário ter dados suficientes para que possa
fazer consultas interessantes.
4) Modele os dados para seu banco de grafos e para seu banco de documentos. Faça
um diagrama para representar sua modelagem em cada banco.
5) Descreva os comandos para inserção dos dados no SGBD e defina o conjunto de
consultas que a aplicação fará no SGBD. Monte, no mínimo, 10 consultas
distintas. Por consultas distintas, leia-se: consultas que tenham operadores/lógicas
diferentes, de forma a mostrar as possibilidades de consulta que o SGBD lhe dá.
A nota será atribuída de acordo com o quão interessante forem as consultas que
você gerou. Então, se gerar apenas consultas simplórias, você poderá ter também
uma nota simplória.
6) Crie um relatório documentando a modelagem, comandos de inserção dos dados,
objetivos de cada consulta, comando da consulta e resultados.
7) Envie o dump de cada um dos bancos + relatório para o Classroom.
